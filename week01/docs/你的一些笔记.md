# Week01 学习笔记

## 本周作业完成情况

- 基础版：完成 `GET /api/hello` 接口。
- 挑战版：完成 `GET /api/health`（应用健康检查）和 `GET /api/system-info`（系统信息）接口。
- 补充了接口自动化测试，确保返回结果符合预期。

## 接口说明

1. `GET /api/hello`
   - 返回示例：`Hello, Spring Boot!`
2. `GET /api/health`
   - 返回字段：`status`、`timestamp`、`uptimeMs`
3. `GET /api/system-info`
   - 返回字段：操作系统、Java 版本、CPU 核数、内存、主机名和 IP 等信息

## 运行与验证

```bash
cd week01/hello-sample
./mvnw spring-boot:run
```

浏览器访问：

- `http://localhost:12348/api/hello`
- `http://localhost:12348/api/health`
- `http://localhost:12348/api/system-info`

## 本周学习小结

这周主要熟悉了 Spring Boot 的基本开发流程：创建控制器、设计 REST 接口、组织返回数据以及本地调试。通过实现健康检查和系统信息接口，我对 Java 获取运行时信息（如 JVM 启动时长、系统属性、内存信息）有了更直观的理解。  
另外，补了 MockMvc 测试后也体会到接口开发不应只看“能跑通”，还要能被自动化验证，后续会继续加强单元测试和异常处理能力。
